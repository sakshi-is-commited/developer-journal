# 💡 Key Learnings

Today's biggest realization was that Git does not simply save files.

Instead, Git stores different types of objects that work together:

* Blob objects store file contents.
* Tree objects represent the project's directory structure.
* Commit objects connect snapshots together to form project history.
* Every object has a unique SHA-1 hash that acts as its identity.
* HEAD is a special reference that indicates the current branch or commit being worked on.

Understanding these concepts made Git feel much less like a collection of commands and more like a structured system.
