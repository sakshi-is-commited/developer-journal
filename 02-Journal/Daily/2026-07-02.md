# 💡 Key Learnings

Today I realized that Git is much more than a collection of commands.

Some important concepts that became clear:

* Git keeps a complete history of a project.
* The Working Directory is where files are edited.
* The Staging Area acts like a preparation area before committing changes.
* A commit is a permanent snapshot of the project at a specific point in time.
* `git status` is one of the most important commands because it shows the current state of the repository.
* Small, meaningful commits make project history easier to understand.

---
