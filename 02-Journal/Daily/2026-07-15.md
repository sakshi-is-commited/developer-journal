# 💡 Key Learnings

## Why Deleting Data Requires More Care Than Updating

Updating modifies existing information while preserving the project itself.

Deleting permanently removes data, making recovery difficult unless backups exist.

Because deletion is irreversible, it requires additional safeguards such as confirmation prompts and careful validation before modifying stored data.

This helped me understand why destructive operations deserve extra attention during software development.

---

## Confirmation Improves Reliability

Today's implementation demonstrated how a simple confirmation dialog can prevent costly mistakes.

Allowing users to review the selected project before confirming deletion reduces accidental removals and improves confidence while using the application.

Many professional applications use the same pattern for operations that permanently remove data.

---

## Defensive Programming

Today's feature introduced me to the concept of defensive programming.

Instead of assuming that users will always make correct decisions, the application now anticipates mistakes by:

- Displaying project details before deletion.
- Asking for confirmation.
- Preventing unintended deletion.
- Handling invalid project selections gracefully.

This approach makes software safer and more reliable.

---

# 🤔 Challenges Faced

One challenge was deciding how users should identify the project they wanted to delete.
