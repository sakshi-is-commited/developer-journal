# 💡 Key Learnings

## Updating Data Is More Complex Than Adding Data

Adding a project simply appends new information to the project list.

Updating a project requires several additional steps:

- Finding the correct project.
- Loading existing data.
- Allowing selective changes.
- Preserving unchanged values.
- Validating updated information.
- Writing the modified project back to storage.

This showed me that modifying existing data involves more decision-making than creating new data.

---

## Improving User Experience

Allowing users to leave fields blank while keeping existing values significantly improves usability.

Users do not need to re-enter information that has not changed, making the update process faster and less error-prone.

I also improved usability by allowing project selection through numbered options instead of requiring users to type the complete project name.

---

## Mutable Data Structures

Today's implementation helped reinforce my understanding of mutable objects in Python.

Since dictionaries inside lists are mutable, I could modify the selected project's values directly before saving the updated list back to the JSON file.

Understanding how mutable data structures behave made implementing the update feature much easier.

---

# 🤔 Challenges Faced

One of the main challenges was deciding how users should select the project they wanted to update.

Initially, I considered searching by project name, but displaying a numbered list proved to be a more user-friendly solution.

Another challenge was ensuring that blank inputs preserved existing values without accidentally replacing valid information with empty strings.

Carefully separating validation, update logic, and storage responsibilities helped keep the implementation clean.

# 📝 Reflection

Today's implementation helped me realize that updating existing data requires much more careful planning than simply creating new data.

Designing the feature to preserve existing values made the application more user-friendly and demonstrated the importance of considering the user's experience during development.

I also appreciated how the modular architecture simplified the implementation. The Storage Layer handled persistence, the Service Layer managed the update logic, and the CLI focused only on interacting with the user.

As more features are added, I can clearly see the benefits of separating responsibilities. New functionality can be introduced without significantly modifying existing code, making the project easier to maintain and extend.

Today's work reinforced that building software is not only about making features work but also about designing them in a way that keeps the code organized, reusable, and pleasant for users.
