# 💡 Key Learnings

Today's biggest learning was understanding the importance of separating responsibilities inside an application.

Instead of writing everything in one file, I organized the project into different modules such as CLI, Services, Models, Storage, and Utilities.

This makes the code easier to understand, maintain, and extend as the application grows.

I also learned how JSON can act as a simple database for small applications before migrating to more advanced databases in future sprints.

---

# 🤔 Challenges Faced

One of the biggest challenges was deciding how to organize the project structure.

Initially, it felt easier to write everything in a single file, but separating functionality into different modules required more planning.

I also spent time ensuring that validation, storage, and business logic remained independent from each other.

Although it required more effort, the resulting architecture is much cleaner.
