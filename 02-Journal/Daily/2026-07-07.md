# 💡 Key Learnings

Today's biggest realization was that Git provides multiple ways to undo mistakes depending on the situation.

I learned that:

- Git Restore is useful for discarding uncommitted file changes.
- Soft Reset removes commits but preserves staged changes.
- Mixed Reset removes commits and unstages the changes while keeping them in the working directory.
- Hard Reset completely removes commits and local changes.
- Git Revert safely undoes changes by creating a new commit without rewriting history.

Understanding the difference between these commands helped me build confidence in recovering from mistakes without panicking.

---

# 🤔 Challenges Faced

Initially, it was difficult to remember how each reset mode affected the commit history, staging area, and working directory.

After performing each command myself and observing the repository state with `git status` and `git log`, the differences became much clearer.

I also realized the importance of thinking carefully before using `git reset --hard` because it permanently discards uncommitted work.
