# 🎯 Today's Objective

Over the last two days, I focused on transforming the **Developer Portfolio Manager** from a functional CRUD CLI application into a **professional, portfolio-ready software project**.

Instead of building new features, I concentrated on improving the project's architecture, user experience, testing, exception handling, and documentation.

This sprint helped me understand that writing software is not only about making it work—it is about making it clean, maintainable, reliable, and easy for others to understand.

## 4. Added Automated Unit Tests

One of today's biggest milestones was introducing automated testing using Python's built-in `unittest` framework.

### Test Cases Implemented

- Add Project
- Search Existing Project
- Search Missing Project
- Update Project
- Delete Project

I executed all tests using:

```bash
python3 -m unittest discover tests
```

### Result


All test cases passed successfully, giving me confidence that the application's core functionality works correctly.

# 🧠 Challenges Faced

While testing the Update Project feature, I encountered an issue where entering an invalid status caused the update to abort.

This reminded me that validation rules must be strict and consistent to maintain data integrity.

I also initially ran the unit tests incorrectly, resulting in:

```text
Ran 0 tests
```

After reviewing the project structure and using the correct unittest discovery command, all tests executed successfully.
